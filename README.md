# ChronOS Insights Local Report Toolkit

This workspace contains a static ChronOS GitHub insights report plus helper scripts to support one-time token setup and local proxy-based API access on Windows.

## Deployment Readiness Status

- Static app architecture: ready (single HTML + optional helper scripts)
- Build step required: none
- Runtime package manager dependencies: none
- Secrets in repository: safe if `.env` remains local and uncommitted
- Entry URL for static hosts: `index.html` (serves the full report directly)
- Browser requirements: modern browser with JavaScript enabled

## File-by-File Explanation

### Core Report
- `ChronalLabs_ChronOS_Report.html`
  - Main interactive report UI.
  - Fetches repository data from GitHub API.
  - Supports two auth modes:
    - Direct browser mode (token pasted into page).
    - Local proxy mode (`http://127.0.0.1:8787`) so token is kept out of the browser input.
  - Can export a generated detailed summary HTML file after refresh.

- `index.html`
  - Primary deployment entrypoint shown at root URL (`/`).
  - Contains the same full report UI so users land directly on the dashboard.

### Hosted Proxy Endpoints (Vercel)
- `api/health.js`
  - Health endpoint for hosted proxy detection.
  - Reports whether server-side environment token is configured.

- `api/github-proxy.js`
  - Server-side pass-through for GitHub API requests.
  - Injects `CHRONOS_GITHUB_TOKEN` (or `GITHUB_TOKEN`) so users do not need to paste tokens.
  - Restricts target URLs to `https://api.github.com/*`.

### Setup and Runtime Scripts
- `chronos-token-setup.ps1`
  - One-time token setup script.
  - Prompts for GitHub token securely.
  - Stores token in either:
    - Local `.env` file (default), or
    - DPAPI-encrypted file (`%USERPROFILE%\.chronos\github_token.dpapi`) when `-UseDpapi` is used.
  - Optionally creates a Startup shortcut when `-CreateStartupShortcut` is provided.

- `chronos-proxy-server.ps1`
  - Starts a local HTTP listener on `127.0.0.1:8787`.
  - Reads token from this priority order:
    1. Process environment variable `GITHUB_TOKEN`
    2. Local `.env` (`GITHUB_TOKEN=...`)
    3. DPAPI token file
  - Exposes:
    - `GET /health` (status endpoint)
    - `GET /api?url=https://api.github.com/...` (token-injected GitHub pass-through)

- `chronos-proxy-start.bat`
  - Convenience launcher for `chronos-proxy-server.ps1`.
  - Useful for quick double-click start on Windows.

### Documentation
- `CHRONOS_TOKEN_ONE_TIME_SETUP.md`
  - Step-by-step setup guide for token storage, proxy startup, and report usage.

### Local Environment Files
- `.env.example`
  - Template showing expected token variable format:
    - `GITHUB_TOKEN=ghp_your_token_here`

- `.env`
  - Local token file generated by setup script (or manually created from template).
  - Contains sensitive credentials and should stay local.

- `.gitignore`
  - Excludes `.env` from version control.

## Typical Workflow

1. Run one-time setup:

```powershell
powershell -ExecutionPolicy Bypass -File .\chronos-token-setup.ps1 -CreateStartupShortcut
```

2. Start proxy:

```powershell
powershell -ExecutionPolicy Bypass -File .\chronos-proxy-server.ps1
```

Or launch with:

```bat
.\chronos-proxy-start.bat
```

3. Open `ChronalLabs_ChronOS_Report.html` in a browser.
4. Leave token input empty if proxy mode is active.
5. Click **Refresh Insights**, then optionally **Download Detailed Report**.

## Dependencies and External Access

### Required for Basic Deployment
- Any static hosting platform (GitHub Pages, Netlify, Vercel, Azure Static Web Apps, S3/CloudFront, etc.)
- Modern browser (Chrome/Edge/Firefox/Safari)
- Outbound network access to:
  - `https://api.github.com`
  - `https://fonts.googleapis.com`
  - `https://fonts.gstatic.com`

### Optional for Local Secure Proxy Mode
- Windows with PowerShell
- Local proxy listening on `http://127.0.0.1:8787`
- Token stored in local `.env` or DPAPI token file

### Not Required
- Node.js
- npm/yarn/pnpm
- Python
- Any backend server for direct browser mode

## Pre-Deployment Checklist

1. Ensure `.env` is present in `.gitignore` (already configured).
2. Confirm no token is hardcoded in HTML/JS.
3. Open report locally and click **Refresh Insights** once.
4. Confirm page loads in mobile width (responsive check).
5. Confirm local proxy mode is optional and direct browser mode still works.
6. Commit only non-secret files.

## Step-by-Step Deployment

### Option A: GitHub Pages (recommended)

1. Push this folder to a GitHub repository.
2. In repository settings, open **Pages**.
3. Set source to your deployment branch (for example `main`) and root `/`.
4. Ensure these files exist at repo root:
  - `index.html`
  - `ChronalLabs_ChronOS_Report.html`
5. Wait for Pages build to finish.
6. Open your published URL and verify:
  - the report opens from root URL,
  - **Refresh Insights** works,
  - **Download Detailed Report** works.

### Option B: Netlify Drag-and-Drop

1. Zip the folder contents (not the parent folder).
2. In Netlify, use **Add new site** -> **Deploy manually**.
3. Drop the zip or folder.
4. Open the generated URL and run the same 3 checks above.

### Option C: Vercel Static Deploy

1. Import the repository in Vercel.
2. Framework preset: **Other**.
3. Build command: leave empty.
4. Output directory: leave empty (root static files).
5. In Vercel Project Settings -> Environment Variables, add one of:
  - `CHRONOS_GITHUB_TOKEN` (recommended), or
  - `GITHUB_TOKEN`
6. Redeploy after saving environment variables.
7. Deploy and validate refresh/download behavior.

## Post-Deployment Validation

1. Load the site in a fresh/private browser session.
2. Verify auth mode hint appears correctly:
  - hosted secure proxy mode on Vercel when env token is configured,
  - direct browser mode on hosted sites,
  - local proxy mode only when local proxy is running.
3. Use one public repo and one private repo test:
  - public repo should refresh without token (subject to rate limits),
  - private repo requires token/proxy with authorized token.
4. Check browser dev tools for failed network calls.
5. Confirm no secret appears in page source or network request URLs.

## Operational Notes

- The report does not auto-refresh on page load in deployment mode; users explicitly trigger **Refresh Insights** to reduce GitHub rate-limit pressure.
- If rate-limited, the report falls back to cached snapshot data (when available).
- On Vercel, the hosted proxy routes are:
  - `/api/health`
  - `/api/github-proxy?url=https://api.github.com/...`
- For team/internal environments, local proxy mode is useful to avoid pasting tokens into browser fields.

## Security Notes

- Keep `.env` private and never commit it.
- Treat generated exported report files as potentially sensitive if they include repository activity data.
- If using DPAPI mode, token can only be decrypted by the same Windows user profile that created it.
